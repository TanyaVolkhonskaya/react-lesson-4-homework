# Домашнее задание: продвинутый React Router

Цель — закрепить работу с React Router, состояниями и контекстом. Нужно расширить учебное приложение и реализовать сценарии с вложенными маршрутами, редиректами и защитой страниц.

## Что нужно сделать

1. **Сделать глобальное хранилище user.**

    - Создать контекст в котором будут храниться юзеры
    - Обернуть все приложение в этот контекст
    - Добавить методы получения юзеров, юзера

2. **Спроектировать профиль студента.**
    - Добавьте маршрут `/users/:userId/profile`, который показывает вкладки: «Инфо», «Домашки» (для них тоже нужны роуты).
    - Данные о студенте требуется брать из глобального контекста.

## Дополнительные идеи (по желанию)

1. **Использовать реальные данные и поработать с сетью.**
    - Выбрать апи, например [jsonplaceholder](https://jsonplaceholder.typicode.com/) или [dummyjson](https://dummyjson.com/docs/users#users-all) (подойдет для тех, кто решил не останавливаться на получении данных). Если там нет нужных для вас полей можете их приклеивать на фронте, например request.then((data) => ({ ...data, favoriteLanguage: <логика рандомного назначения> })). В этом примере я приклеил к данным поле favoriteLanguage
    - Для реализации вам потребуется использовать `useEffect` и какие-нибудь самописные хуки в духе `useFetch` (написать его, там будет логика получения данных из фетча в стейт, там же можно сделать isLoading), хуки `useUser` и `useUsers` которые будут получать данные
    - В идеале сделать это как-то глобально, чтобы данные запрашивались один раз и лежали в глобальном контексте
2. **Работа с поиском и фильтрами.**
    - На странице `/users` сделайте форму поиска по имени и/или любой фильтр/сортировку.
        > Если брать dummyjson, то там уже есть фильтры, можно их взять и реализовать
    - Состояние фильтров храните в URL (`/users?role=student&query=...`). Используйте `useSearchParams`.

## Идеи для самых смелых (супер по желанию)

1. **Сделать авторизацию и защищенные роуты**
    - Добавить роут для логина
    - Написать авторизацию (можно на фронте без бека, а можно используя dummyjson(рекомендую попробовать это, тк это уже прям кейс из реальных проектов))
    - Сделать какие-то роуты защищенными, например чтобы нельзя просматривать никакие страницы кроме лендинга на роуте `/`. Для этого надо сделать 2 лейаута `PublicRoutes` `PrivateRoutes` и если человек неавторизован, то он сможет видеть только `PublicRoutes`, а при переходе на `PrivateRoutes` будет редирект на страницу `/login`
